Bootstrap: Definition

bootstrap: an approach that approximates the sampling distribution of an estimator by repeatedly resampling from the observed data with replacement

Bootstrap Approach:

  • treats the observed dataset as a proxy for the population
  • generates many “new” samples by sampling with replacement
  • computes the estimator in each resample
  • uses the resulting distribution to quantify variability

Bootstrap: Definition

Goals within statistical estimation include:

  • how much an estimate varies across samples
  • how precise the estimate is
  • what values are plausible for the population

Approaches rely on:

  • known sampling distributions
  • analytic standard error formulas
  • assumptions about the data [i.e.: normality]

However in certain cases: assumptions do not align with observed data

  • closed-form methods for estimating uncertainty rely on assumptions about the data are true in order to estimate sample variability [i.e.: normality and simple variance structure]

  • bootstrap methods estimate uncertainty by repeatedly resampling the data and observing how the estimate changes

Overview: Bootstrap Procedure

Given observed data of size n:

1. Resample with replacement

  • draw a sample of size n from the original data

2. Compute the estimator - i.e.: mean, median, risk difference

3. Repeat many times - i.e.: 1,000+ resamples

4. Form the bootstrap distribution - collection of all computed estimates

Resulting distribution approximates the sampling distribution of the estimator

Overview: Bootstrap Procedure

From the bootstrap distribution, we can estimate:

  1. Standard error: measures variability across resamples

  2. Confidence intervals: provides a plausible range for the population parameter

  3. Shape of the sampling distribution :

  • symmetry vs skewness
  • sensitivity to outliers
  • stability of the estimator

Example: Estimating Typical Tumor Mutation Burden

Background

Investigators are studying tumor genomic profiles from patients with solid tumors.

The quantitative genomic measure of interest is tumor mutation burden, or TMB.

Tumor mutation burden is the number of somatic mutations observed per megabase of sequenced tumor DNA.

TMB provides a summary of how many mutations are present in a tumor genome and is commonly used to describe variation in tumor genomic profiles across patients.

In this study, TMB is measured from tumor sequencing data to summarize the mutational profile of the patient cohort.

Because TMB values are often right-skewed:

  • many tumors have low-to-moderate mutation burden
  • fewer tumors have very high mutation burden
  • extreme values can pull the mean upward

Investigators want to estimate:

What is a typical tumor mutation burden among patients in this sequencing cohort?

Bootstrap Appraoch

bootstrap resampling: approximates repeated sampling using the observed data

Each bootstrap iteration executes the following:

1. resample TMB values with replacement

  • after a value is selected, it is returned to the dataset and can be selected again
  • allowing the bootstrap sample to:
    • maintain same size as the original dataset
    • note that each resample may include some participants more than once and may omit some participants

2. compute the estimator on that resampled dataset

3. store the result

4. repeat many times

After many iterations the collection of estimates approximates the sampling distribution of the estimator

Bootstrap for the Mean TMB

set.seed(222)

boot_means <- replicate(2000, {

  x_star <- sample(

  mean(x_star)

Estimating the Bootstrap Standard Error

boot_mean_se <- sd(boot_means)
boot_median_se <- sd(boot_medians)

tibble(
  estimator = c("Mean TMB", "Median TMB"),
  bootstrap_se = c(boot_mean_se, boot_median_se)
)
# A tibble: 2 × 2
  estimator  bootstrap_se
  <chr>             <dbl>
1 Mean TMB          0.310
2 Median TMB        0.369
  • bootstrap standard error is computed directly from the bootstrap estimates
  • measures how much the estimator varies across repeated resamples

\[ SE_{\mathrm{boot}} = \sqrt{ \frac{1}{B - 1} \sum_{b=1}^{B} (\theta_b^* - \bar{\theta}^*)^2 } \] Where:

  • \(B\) : number of bootstrap resamples
  • \(\hat{\theta}_b\) : estimator from bootstrap sample \(b\)
  • \(\bar{\hat{\theta}}\) : average bootstrap estimate

Bootstrap Confidence Interval

mean_ci <- quantile(
  boot_means,
  probs = c(0.025, 0.975)
)

median_ci <- quantile(
  boot_medians,
  probs = c(0.025, 0.975)
)

tibble(
  estimator = c("Mean TMB", "Median TMB"),
  lower_95_ci = c(mean_ci[1], median_ci[1]),
  upper_95_ci = c(mean_ci[2], median_ci[2])
)
# A tibble: 2 × 3
  estimator  lower_95_ci upper_95_ci
  <chr>            <dbl>       <dbl>
1 Mean TMB          7.04        8.25
2 Median TMB        5.93        7.43

Interpretation

  • confidence intervals contain the middle 95% of estimates generated from repeated bootstrap resampling

For a 95% bootstrap confidence interval:

  • lower bound is the 2.5th percentile of the bootstrap estimates

  • upper bound is the 97.5th percentile of the bootstrap estimates

  • each bootstrap estimate represents one possible estimate of population TMB obtained from a resampled version of the observed sequencing dataset

  • some bootstrap estimates are smaller and some are larger because sampling variability causes estimates to fluctuate across samples

  • confidence interval summarizes the range of population TMB values that remain plausible given the observed data and the observed variability across resamples

Summary

  • point estimates summarize characteristics of a population using observed sample data

  • different samples produce different estimates because of sampling variability

  • bootstrap resampling approximates repeated sampling directly from the observed dataset

  • bootstrap methods repeatedly:

    • resample the data with replacement
    • recompute the estimator
    • form an empirical sampling distribution

bootstrap distributions can be used to estimate:

  • standard errors
  • confidence intervals
  • estimator stability
